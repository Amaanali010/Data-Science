# PART 2 — What is Big Data?

## Definition

Big Data refers to extremely large and complex datasets that cannot be efficiently managed using traditional database systems.

## Simple Explanation

Big Data is:

- Huge in size
- Generated rapidly
- Comes from multiple sources
- Difficult to process using conventional tools

## Sources of Big Data

### 1. Social Media
- Facebook posts
- Instagram photos
- TikTok videos

### 2. IoT Devices
- Smart watches
- Sensors
- Smart homes

### 3. E-Commerce
- Amazon purchases
- Online browsing history

### 4. Banking Systems
- ATM transactions
- Online banking

### 5. Healthcare
- Patient records
- Medical imaging

## Why is Big Data Important?

Companies use Big Data to:

- Improve decision-making
- Understand customers
- Predict trends
- Increase profits
- Detect fraud

# PART 3 — Characteristics of Big Data (5 Vs)

## 1. Volume

Refers to the enormous amount of data.

### Examples
- Terabytes
- Petabytes
- Exabytes

### Real Example
Netflix stores massive viewing data from millions of users.

---

## 2. Velocity

The speed at which data is generated and processed.

### Examples
- Live stock market data
- Social media feeds
- GPS tracking

### Discussion
Why is real-time processing important in banking fraud detection?

---

## 3. Variety

Different forms of data.

### Structured Data
- Excel sheets
- SQL databases

### Semi-Structured Data
- JSON
- XML

### Unstructured Data
- Images
- Videos
- Audio
- Emails

---

## 4. Veracity

Accuracy and reliability of data.

### Problems
- Duplicate data
- Missing values
- Incorrect information

### Example
Incorrect customer data may lead to poor business decisions.

---

## 5. Value

Useful insights generated from data.

### Examples
- Product recommendations
- Fraud detection
- Disease prediction

## Activity

Ask students:

> “Identify the 5 Vs in TikTok or YouTube.”

# PART 4 — Traditional Databases vs Big Data

| Feature | Traditional Databases | Big Data Systems |
|---|---|---|
| Data Size | Small/Medium | Very Large |
| Data Type | Structured | Structured + Unstructured |
| Processing | Centralized | Distributed |
| Scalability | Limited | Highly Scalable |
| Speed | Moderate | High-Speed |

## Key Concept

Traditional systems struggle with:

- Large-scale storage
- Fast processing
- Unstructured data

Big Data systems solve these using distributed computing.

# PART 5 — Big Data Technologies

# 1. Hadoop

## Definition

An open-source framework for distributed storage and processing.

## Hadoop Components

### HDFS
Hadoop Distributed File System

- Stores data across multiple machines
- Fault tolerant

### MapReduce
Processes large data in parallel.

## Advantages

- Scalable
- Cost-effective
- Reliable

---

# 2. Apache Spark

## Definition

A fast distributed processing engine.

## Features

- In-memory processing
- Faster than Hadoop MapReduce
- Supports machine learning

## Applications

- Real-time analytics
- Streaming
- AI systems

---

# 3. NoSQL Databases

## Examples

- MongoDB
- Cassandra
- HBase

## Why NoSQL?

Traditional SQL databases struggle with flexible and massive datasets.

## Features

- Flexible schema
- Horizontal scaling
- High performance
